# Деплой на Яндекс Object Storage

Сайт хостится на Яндекс Object Storage. При каждом пуше в `main` GitHub Actions автоматически заливает файлы в бакет.

URL сайта: `https://kazan-music-history.website.yandexcloud.net`

---

## Первоначальная настройка

### 1. Яндекс Облако — создать аккаунт

1. Идёшь на [console.yandex.cloud](https://console.yandex.cloud) и логинишься через Яндекс-аккаунт
2. При первом входе нажимаешь **"Создать организацию"** — название любое
3. Привязываешь карту (снимут 1₽ для проверки, потом вернут)

### 2. Создать бакет

1. В левом меню → **Object Storage** → **"Создать бакет"**
2. Заполняешь:
   - **Имя бакета:** `kazan-music-history`
   - **Доступ на чтение объектов:** `Публичный`
   - **Доступ к списку объектов:** `Публичный`
3. Нажимаешь **"Создать бакет"**

### 3. Включить хостинг на бакете

1. Открываешь бакет → вкладка **"Веб-сайт"**
2. Переключаешь на **"Хостинг"**
3. **Главная страница:** `index.html`
4. **Страница ошибки:** `index.html`
5. Нажимаешь **"Сохранить"**

### 4. Создать сервисный аккаунт

1. В левом меню → **"Сервисные аккаунты"** → **"Создать сервисный аккаунт"**
2. Имя: `github-deploy`
3. Роль: `storage.editor`
4. Нажимаешь **"Создать"**
5. Открываешь аккаунт → **"Ключи доступа"** → **"Создать статический ключ доступа"**
6. Сохраняешь оба значения — потом не покажет:
   - `Идентификатор ключа` (Access Key ID)
   - `Секретный ключ` (Secret Access Key)

### 5. Добавить секреты в GitHub

1. GitHub → репозиторий → **Settings** → **Secrets and variables** → **Actions**
2. Добавляешь два секрета:

| Name                   | Value                         |
| ---------------------- | ----------------------------- |
| `YC_ACCESS_KEY_ID`     | Идентификатор ключа из шага 4 |
| `YC_SECRET_ACCESS_KEY` | Секретный ключ из шага 4      |

После этого всё готово — при каждом пуше в `main` сайт обновляется автоматически.

---

## Как работает пайплайн?

Файл `.github/workflows/deploy.yml` при пуше в ветку `main`:

1. Берёт файлы из репозитория
2. Синхронизирует их в бакет через AWS CLI (Яндекс Object Storage совместим с S3)
3. Файлы, которых нет в репозитории, удаляются из бакета (`--delete`)

Прогресс деплоя: вкладка **Actions** на GitHub.
